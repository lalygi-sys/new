# Disko — Product Discovery Backlog

> Бэклог ведётся по методологии Quadruple Diamond.
> Статус каждой инициативы отражает текущий октант прохождения.

## Прогресс дискавери

```
[□□□□□□□□] 0/8 — Ожидание старта сессии с PM
```

---

## 🔷 Diamond 1: Framing the Universe

### Octant 1 — Exploration (дивергенция)
> Статус: ⏳ Ожидает

**Цель:** собрать максимум контекста о продукте, бизнесе и пользователях.

- [ ] Бизнес-контекст (продукт, модель, этап)
- [ ] Стратегические цели и OKR
- [ ] Известные боли клиентов (топ-3)
- [ ] Ресурсы и ограничения
- [ ] Загрузка существующих артефактов
- [ ] Исследование рынка и конкурентов
- [ ] SWOT-анализ
- [ ] Landscape Map

### Octant 2 — North Star (конвергенция)
> Статус: ⏳ Ожидает

**Цель:** выбрать главный приоритет, сформулировать North Star.

- [ ] Аффинити-группировка проблем
- [ ] HMW-вопросы
- [ ] Приоритизация с PM
- [ ] North Star Statement + Metric
- [ ] Roadmap (Now / Next / Later / Won't)

---

## 🔷 Diamond 2: Designing the Right Thing

### Octant 3 — Targeted Expansion (дивергенция)
> Статус: ⏳ Ожидает

**Цель:** deep dive в проблемное пространство.

- [ ] Customer Journey (as-is)
- [ ] Данные и метрики
- [ ] Конкурентный анализ (Competitive Matrix)
- [ ] Root Cause Analysis
- [ ] Бенчмарки

### Octant 4 — Setting the Course (конвергенция)
> Статус: ⏳ Ожидает

**Цель:** определить что строим, MVP scope.

- [ ] Список инициатив
- [ ] RICE-скоринг
- [ ] MoSCoW приоритизация
- [ ] MVP Backlog с гипотезами

---

## 🔷 Diamond 3: Designing the Thing Right

### Octant 5 — Ideation & Prototyping (дивергенция)
> Статус: ⏳ Ожидает

**Цель:** для каждого Must-элемента — минимум 3 варианта решений.

- [ ] Варианты A/B/C для каждого Must
- [ ] Исследование паттернов решений
- [ ] Feasibility scoring
- [ ] Оценка от команды

### Octant 6 — Concept Validation (конвергенция)
> Статус: ⏳ Ожидает

**Цель:** выбрать лучший вариант, валидировать через бенчмарки.

- [ ] Scoring Matrix
- [ ] Валидация через аналоги
- [ ] Финальное решение PM
- [ ] GO / NO-GO

---

## 🔷 Diamond 4: Delivering the Right Thing

### Octant 7 — Development & Deployment (дивергенция)
> Статус: ⏳ Ожидает

**Цель:** декомпозировать решение на поставляемые user stories.

- [ ] User Stories с Acceptance Criteria
- [ ] T-shirt estimation
- [ ] Порядок и зависимости
- [ ] Release Plan

### Octant 8 — Value Realisation (конвергенция)
> Статус: ⏳ Ожидает

**Цель:** метрики успеха и план измерения.

- [ ] Primary Metrics + Guard Rails
- [ ] Measurement Plan
- [ ] Experiment Design (A/B / rollout)
- [ ] Decision Framework (scale / iterate / stop)
