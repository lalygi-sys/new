### LANDSCAPE MAP: IB Contest Platform

МОДЕЛЬ: Платформенная — партнёр зарабатывает комиссию от объёма торгов клиентов; контесты стимулируют рост объёма
ЭТАП: Запуск (0 → 1)

---

### БИЗНЕС-КОНТЕКСТ

**Как работает IB-модель:**
- IB (Introducing Broker) привлекает клиентов и получает комиссию с каждого отторгованного лота
- Чем больше клиенты торгуют (объём + частота), тем больше зарабатывает IB
- Контесты — инструмент IB для стимулирования активности и удержания клиентов

**Текущее состояние (as-is):**
- Партнёры проводят контесты вручную: Excel-таблицы, Telegram-каналы, Google Sheets
- Подсчёт результатов — ручной запрос данных через поддержку или API
- Объявление победителей — посты в соцсетях/мессенджерах
- Выплата призов — ручной перевод на торговый счёт или бонусы
- Нет единого интерфейса, нет прозрачности, нет автоматизации

**Как партнёры проводят контесты сейчас (реконструкция процесса):**
1. Придумывают правила → пишут пост в Telegram/WhatsApp группу
2. Клиенты «записываются» в комментариях или через Google Form
3. В течение контеста IB вручную запрашивает данные по объёмам
4. По итогам — сводит результаты в Excel, публикует таблицу
5. Выплачивает призы через запрос в бэк-офис или лично
6. Весь процесс занимает много времени и подвержен спорам о честности

### ЦЕЛИ КОМПАНИИ

- Увеличить торговый объём через активацию IB-канала
- Повысить retention клиентов IB-партнёров
- Дать партнёрам self-serve инструмент (снизить нагрузку на бэк-офис)
- Создать конкурентное преимущество (ни один брокер не предлагает self-serve контесты для IB)

### SWOT-АНАЛИЗ

| Strengths | Weaknesses |
|-----------|------------|
| Прямой доступ к торговым данным (объёмы, лоты) | Нет существующей платформы — строим с нуля |
| IB-партнёры уже проводят контесты (есть спрос) | Нет исторических данных по эффективности контестов |
| Уникальное предложение на рынке | Зависимость от интеграции с торговым бэкендом |
| Win-win: контесты увеличивают и доход IB, и revenue брокера | Нужна модерация для предотвращения фрода |

| Opportunities | Threats |
|--------------|---------|
| Ни один конкурент не предлагает self-serve контесты для IB | Регуляторные ограничения на промо в некоторых юрисдикциях |
| Геймификация — тренд в fintech (лидерборды, streaks, badges) | Фрод: wash trading, мульти-аккаунты, bonus abuse |
| Данные о контестах = инсайты для продуктовых решений | Партнёры могут злоупотреблять (обещать нереальные призы) |
| Экспорт модели на других партнёров/рынки | Сложность UX — партнёры не технические пользователи |

### ПРОБЛЕМЫ (ранжированы по серьёзности)

1. **Ручной подсчёт результатов** — severity: высокая — источник: реконструкция процесса
   IB тратит часы на сбор и сверку данных, что делает контесты невыгодными по времени

2. **Отсутствие доверия к результатам** — severity: высокая — источник: индустрия
   Клиенты не видят свой прогресс в реальном времени, не доверяют ручному подсчёту

3. **Нет инструмента коммуникации условий** — severity: средняя — источник: процесс
   Правила передаются в чатах, клиенты присоединяются неформально, нет юридической обёртки

4. **Сложность выплаты призов** — severity: средняя — источник: процесс
   Ручной процесс через бэк-офис, задержки, отсутствие аудит-трейла

5. **Нет аналитики эффективности** — severity: средняя — источник: бизнес
   Партнёры не знают ROI контестов, компания не видит влияние на метрики

6. **Фрод-риски** — severity: средняя — источник: индустрия
   Wash trading, мульти-аккаунты, сговор — нет автоматической детекции

---

### ВОЗМОЖНОСТИ

1. **Self-serve конструктор контестов** — потенциал: высокий — источник: gap анализ
   Ни один брокер не предлагает. Категорийный дифференциатор.

2. **Real-time лидерборд** — потенциал: высокий — источник: best practices (OctaFX, XM)
   Решает проблему доверия + повышает engagement

3. **Автоматические выплаты** — потенциал: средний — источник: процесс
   Призы начисляются автоматически на торговый счёт

4. **Шаблоны контестов** — потенциал: средний — источник: аналоги (Gleam, Woobox)
   Снижает порог входа для менее технических партнёров

5. **Аналитика ROI контестов** — потенциал: средний — источник: бизнес
   Партнёр видит, как контест повлиял на объём и количество активных клиентов

### АФФИНИТИ-ГРУППИРОВКА ПРОБЛЕМ

**Кластер A — Операционная неэффективность:**
- Ручной подсчёт результатов
- Сложность выплаты призов
- Нет инструмента коммуникации условий

**Кластер B — Доверие и прозрачность:**
- Отсутствие real-time прогресса
- Споры по результатам
- Нет аудит-трейла

**Кластер C — Бизнес-аналитика:**
- Нет данных об эффективности контестов
- Нет связи "контест → рост объёма"
- Компания не видит влияние на retention

**Кластер D — Безопасность:**
- Фрод-риски (wash trading, мульти-аккаунты)
- Нет модерации
- Нет compliance framework

### HMW-ВОПРОСЫ (ранжированы)

1. **How might we** дать IB-партнёрам возможность самостоятельно запускать контесты за минуты, а не дни, чтобы они проводили контесты в 5x чаще? — Impact: 5, Effort: 4
2. **How might we** обеспечить прозрачный real-time прогресс для участников, чтобы доверие к результатам было 100%? — Impact: 5, Effort: 3
3. **How might we** автоматизировать выплату призов, чтобы исключить ручную работу и задержки? — Impact: 4, Effort: 3
4. **How might we** дать компании видимость в эффективность контестов, чтобы принимать data-driven решения? — Impact: 4, Effort: 2
5. **How might we** предотвратить фрод в контестах, чтобы результаты были честными без ручной проверки? — Impact: 3, Effort: 4

---

### NORTH STAR

**Statement:** Каждый IB-партнёр может запустить прозрачный торговый контест для своих клиентов за 5 минут — от настройки правил до автоматической выплаты призов.

**North Star Metric:** Количество успешно завершённых контестов в месяц
**Baseline:** 0 → **Target:** 50 контестов/мес (через 6 мес после запуска)

**Поддерживающие метрики:**
- Среднее время создания контеста: < 5 минут
- % контестов без споров по результатам: > 95%
- Рост торгового объёма участников контестов vs non-participants: +30%

### ROADMAP

**NOW (MVP, 1-2 месяца):**
- Конструктор контестов (wizard) для IB
- Real-time лидерборд для участников
- Базовая admin-панель для модерации
- Автоматический подсчёт результатов (объём, streaks)

**NEXT (2-4 месяца):**
- Автоматическая выплата призов на торговый счёт
- Шаблоны контестов (Quick Start)
- Аналитика ROI для партнёра
- Фрод-детекция (wash trading rules)

**LATER (4-6 месяцев):**
- Виджет для встраивания лидерборда на сайт IB
- Push-уведомления участникам
- Программа лояльности (badges, tiers, streaks across contests)
- A/B тестирование механик контестов

**WON'T (осознанно не делаем сейчас):**
- Социальная лента / feed (слишком сложно, не core value)
- Demo-контесты (фокус на реальной торговле)
- Партнёрский маркетплейс контестов (premature)
