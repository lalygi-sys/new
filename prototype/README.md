# Disko Prototype

Интерактивный прототип платформы контестов для IB-партнёров, клиентов и админов. Один файл [`index.html`](./index.html) с React + JSX (через Babel-standalone из CDN) и встроенной реализацией компонентов RIG/DS.

Прототип используется продуктовой командой для проверки гипотез, демо и согласования флоу до разработки.

## Как запустить у себя

### 1. Склонировать репозиторий

```bash
git clone https://github.com/tkapkaeva-create/disko.git
cd disko
```

### 2. Поднять локальный сервер

Прототип — это статический HTML, он требует только любой http-сервер (браузер не отдаст JSX с `file://`). Любой из вариантов подойдёт:

```bash
# Python 3 (есть из коробки на macOS/Linux)
cd prototype && python3 -m http.server 9130

# или Node.js
cd prototype && npx serve -l 9130

# или VS Code / Cursor — расширение Live Server → Open with Live Server
```

Открыть в браузере: <http://localhost:9130/index.html>

### 3. Работа с прототипом

- Переключение персон — в верхней навигации: **IB Partner / Client / Admin**
- Горячая перезагрузка не встроена; после правок просто обновите страницу (`⌘R` / `Ctrl+R`)
- Консоль показывает предупреждение Babel — это нормально, прототип использует in-browser transformer

## Структура проекта

```
prototype/
├── index.html     # единственный файл: весь CSS, SVG-иконки и React-компоненты
├── README.md      # этот файл: инструкция + changelog
└── CHANGELOG.md   # подробный журнал изменений по комитам
```

Дизайн-система (иконки, переменные, компоненты) — вынесена в `.fig`-файлы в корне репозитория:
- `Icons [1 brand].fig`
- `Foundations.fig`, `Components.fig`, `Guides.fig`, `Graphic assets [1 brand].fig`

## Как вносить изменения

1. Создайте ветку от `main`:
   ```bash
   git checkout -b feat/короткое-описание
   ```
2. Правьте `prototype/index.html` (компоненты, стили, SVG-спрайт — всё внутри).
3. Проверьте в браузере, что ничего не сломалось на всех трёх персонах.
4. Обновите [`CHANGELOG.md`](./CHANGELOG.md) — добавьте запись сверху с датой, хэшем комита (можно вписать после `git commit`) и списком изменений по областям (иконки / визард / клиент / админ / ...).
5. Закоммитьте и запушьте:
   ```bash
   git add prototype/index.html prototype/CHANGELOG.md
   git commit -m "feat(prototype): …"
   git push
   ```

### Соглашение для комитов

- `feat(prototype): ...` — новый функционал или UI
- `fix(prototype): ...` — правка бага
- `chore(prototype): ...` — рефакторинг, переименования, чистка
- `docs(prototype): ...` — README / CHANGELOG

### Как поддерживать CHANGELOG.md

Запись в CHANGELOG.md — обязательна при любом изменении, влияющем на UX или визуальный слой прототипа. Шаблон:

```markdown
## 2026-05-01 — короткое название · [`abc1234`](https://github.com/tkapkaeva-create/disko/commit/abc1234)

### Что изменилось
- **Визард создания контеста:** добавлена вкладка «Призы»
- **Клиент:** показываем актуальный баланс в карточке аккаунта

### Зачем
Кратко: какую задачу/гипотезу это решает.
```

Если не уверены — дублируйте формат последней записи и добавьте свою сверху.
